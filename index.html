spring-simple-web

This simple web (war) example demonstrates how Spring container is configured, instantiated and being used in a web application.

Spring container is initialized here in a traditional way, within WEB-INF/web.xml file, using <listener> element of type org.springframework.web.context.ContextLoaderListener. This listener initializes org.springframework.web.context.ContextLoader instance.

The listener and loader are configured using contextConfigLocation context parameter. The default value for the contextConfigLocation is WEB-INF/applicationContext.xml. However, in this application we used classpath:applicationContext.xml which resides in src/main/resources directory in this Maven web project.

Note that you could use Ant-style path patterns as well, e.g. WEB-INF/spring-*.xml,WEB-INF/*Context.xml.
